### Session [2] — Three-Tier AI Context Model & Section Context Maps

**Context injected:**
- `ai/context_map.md`
- `ai/log_summary.md`

**Goal:** Design a smarter AI context injection system to avoid wasting tokens on full section files when a compressed summary suffices.

**Key conclusions:**
- A three-tier model (Tier 1: always, Tier 2: section context map, Tier 3: full spec) balances token cost against depth
- Section context maps live in `ai/sections/` — separate from content files, created on demand (not upfront for all 20)
- Each section context map captures: what the section specifies, locked decisions, open questions, compliance implications, cross-references

**Decisions made:**
- `ai/sections/` folder created with `_template.md`
- First two section context maps created: `06_dynamic_reconfiguration.md` and `20_dynamic_swapping.md`
- `ai/INSTRUCTIONS.md` updated with the three-tier model and session start/end protocols
